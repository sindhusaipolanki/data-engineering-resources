## 2026-07-13 — SQL: Use GENERATE SERIES to fill date gaps

Reports with gaps in time series confuse business users. Use GENERATE_SERIES (PostgreSQL/Synapse) or a calendar table to produce all dates, then LEFT JOIN your data — gaps show as 0 instead of missing rows.

```sql
-- Fill daily gaps for a 30-day report
WITH calendar AS (
  SELECT generate_series(
    DATE_TRUNC('day', NOW()) - INTERVAL '30 days',
    DATE_TRUNC('day', NOW()),
    INTERVAL '1 day'
  )::date AS report_date
)
SELECT c.report_date, COALESCE(SUM(s.amount), 0) AS total
FROM calendar c
LEFT JOIN sales s ON s.sale_date = c.report_date
GROUP BY c.report_date
ORDER BY c.report_date;
```

## 2026-07-14 — Delta Lake: Set auto-optimize properties on high-write tables

For tables with frequent small writes (streaming or incremental loads), enable auto-optimize. It automatically compacts small files after every write, preventing the small-file problem without a separate OPTIMIZE job.

```sql
ALTER TABLE silver.events
SET TBLPROPERTIES (
  'delta.autoOptimize.optimizeWrite' = 'true',
  'delta.autoOptimize.autoCompact'   = 'true'
);
```

## 2026-07-15 — Data Quality: Use checksums to validate file transfers

When moving large files between storage accounts or cloud regions, always validate with MD5/SHA256 checksums. A corrupted file that passes row count checks but fails checksum is a silent data corruption risk.

```python
import hashlib

def file_checksum(path: str) -> str:
    h = hashlib.md5()
    with open(path, 'rb') as f:
        for chunk in iter(lambda: f.read(8192), b''):
            h.update(chunk)
    return h.hexdigest()

# Compare source and destination checksums
assert file_checksum(source) == file_checksum(dest), 'Checksum mismatch!'
```
